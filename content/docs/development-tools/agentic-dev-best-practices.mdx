## Key Takeaways

If you only remember a few things from this guide:

1. **Write tests first.** Codex was trained to iterate until tests pass. Without tests, it judges its own work — and it's a generous grader.
2. **One task per session.** Codex's cloud <Def>sandbox</Def> model rewards focused, discrete tasks. Start fresh rather than accumulating context.
3. **AGENTS.md is your lever.** A lean, specific configuration file is the primary way to shape Codex's behavior.
4. **Review everything.** Over 40% of AI-generated code contains security flaws. The agent proposes; you decide.
5. **Plan before you prompt.** Write a spec, break it into tasks, then hand each task to Codex. "Prompt to ship" is the fastest way to get into trouble.
